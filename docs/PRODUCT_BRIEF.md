# OpenSaga Product Brief

## One-Sentence Positioning

OpenSaga is an open-source collaborative universe-building platform where
communities create worlds, submit lore, vote on proposals, and preserve shared
canon.

## Problem

Fandom and creator communities often build lore across chats, documents, wikis,
and scattered images. Ideas grow quickly, but canon becomes hard to govern:
duplicates appear, character rules conflict, world history drifts, and the
source of truth becomes unclear.

## Audience

- Anime, manga, comic, and tabletop creators.
- Writing groups building shared worlds.
- Fandom communities managing collaborative canon.
- Open-source contributors interested in creative tooling and governance.

## Use Cases

| Use Case | OpenSaga Helps With |
| --- | --- |
| World creation | Start a world with identity, tone, factions, and rules. |
| Proposal workflow | Submit characters, lore, places, events, or systems. |
| Community voting | Move contributions through proposal, vote, and canon states. |
| World Bible | Preserve accepted truth in a structured source of truth. |
| Creator Studio | Generate and refine character, world, lore, and DNA concepts. |
| Open contribution | Let the community improve the platform transparently. |

## Solution

OpenSaga combines a React application, Supabase-backed data model, governance
workflow, Creator Studio, BYOK AI adapters, local/offline fallback, and
open-source contribution process. The product keeps the platform open while
protecting user-created worlds as creator-owned content.

## Product Boundaries

- OpenSaga is open-source platform code; user-created worlds belong to creators.
- AI is assistive and reviewable, not the owner of canon.
- The current app is pre-public-beta and still needs hosted Supabase setup,
  OAuth, RLS review, deployment, accessibility work, and broader E2E coverage.

## Current Readiness

OpenSaga is on an alpha production track and is the only project in this suite
intended to be open source right now.
