# Support

OpenSaga is currently an alpha open-source project. The best support path depends on what you need.

## Questions

Use GitHub issues for clear, project-related questions that could help future contributors. Before opening an issue, check:

- `README.md`
- `ARCHITECTURE.md`
- `docs/core-concepts.md`
- `docs/deployment.md`
- Existing issues

## Bugs

Use the bug report issue template. Include steps to reproduce, browser/OS details, and whether you are running demo mode or a configured Supabase project.

## Feature Ideas

Use the feature request template. Explain the creator/community problem first, then the proposed solution.

## Security Issues

Do not open public issues for vulnerabilities, leaked secrets, auth bypasses, RLS bypasses, or XSS reports. Follow `SECURITY.md`.

## Conduct Issues

Follow `CODE_OF_CONDUCT.md`. Do not put sensitive personal information in public issues.
